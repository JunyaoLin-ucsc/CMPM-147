Description

This experiment creates "evolutionary impressions" of selected images. A generative algorithm uses simple shapes and iteratively refines its design to visually match a target image, driven by a pixel-difference score and user-controlled mutation rate.

Technical

This generator is powered by my_design,.js and sketch.js, this project generates art by evolving designs of 100 semi-transparent rectangles. getInspirations provides image sources. initDesign sets up initial random parameters (background, shape properties) and handles canvas sizing. renderDesign draws the rectangles using their stored fill and a fixed alpha. mutateDesign applies Gaussian-scaled random changes to these parameters based on a UI slider rate. evaluate uses Mean Squared Error (MSE) pixel comparison against a target image to score designs, with an exponential function mapping error to fitness. The p5.js draw loop drives the evolution.

Reflection

This is a new p5.js technique to me, which is very fun. I was keep getting fascinated by new ways to "draw" by code. After the work, I felt that there is still a lot of place to work on and improve, just like any other experiment. For example, currently the rectangle is the most fit shape for generation based on my testing. However, I will try to discover a more appropriate use of ellipse.
